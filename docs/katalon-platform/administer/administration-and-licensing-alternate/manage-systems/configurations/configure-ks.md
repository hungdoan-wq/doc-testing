This document shows you how to configure Katalon Studio settings in Katalon TestOps.

:::caution requirements
You must have the **Account Admin** or **System Admin** role to perform this action.
:::
## Configure AI Tagging settings

Tags in Katalon Studio help organize and manage test cases. To streamline this process, an **Account Admin** or **System Admin** can configure automatic tagging for test cases created or modified using AI features.

**Auto-tagging is supported for the following actions:**
- API Test Case Generation  
- StudioAssist Code Generation

1. Navigate to **Admin Settings > System > Configurations**.
2. Switch to the **Katalon Studio** tab.
<img src="https://tw-cdn.katalon.com/katalon-platform/configuration/go-to-ks-tab.png" alt="Navigate to KS Configuration tab" width="700"/> <br/>

3. Toggle **Enable auto-tagging for AI usage** to enable or disable the feature.
4. Select the corresponding checkbox for each action to enable tagging. Then, enter a custom tag name that will be automatically applied to test artifacts generated by that action.
5. Click **Save Changes**.
<img src="https://tw-cdn.katalon.com/katalon-platform/configuration/tag-enable.png" alt="KS Config UI - TestOps" width="1080"/>

#### Result
The tag will be automatically applied to your test case in Katalon Studio under **Properties > Tag** once the test is generated using StudioAssist or API generation.

<img src="https://tw-cdn.katalon.com/katalon-platform/configuration/tag-ks.png" alt="Tags in KS" width="1080"/>

:::info note
You can allow individual Katalon Studio users to manage auto-tagging by toggling **Auto-tagging is managed by individual KS users**.
:::

## Set Katalon Studio Idle Time-out

You can manage user session behavior in Katalon Studio by setting idle time-out configurations.

1. In the **System > Configurations > Katalon Studio** tab.
2. - Toggle **Automatically log out inactive users**.
   - Choose a time-out limit from the dropdown (e.g., 1 hour).
3. Click **Save Changes**.
### Allow Users to Extend Session After Timeout
You can configure session time-out behavior and allow users to extend their session a limited number of times:

1. Toggle **Automatically log out inactive users** to enable idle time-out.
    
    Set the Time-out limit (minutes/hours) from the dropdown — this defines how long a user can be inactive before being logged out.
2. Toggle **Allow user to continue after the timeout** to enable the bypass feature.
    
    Enter the number of times each user is allowed to bypass the idle time-out (i.e., how many times they can stay logged in after inactivity).
3. Click **Save Changes**.

<img src="https://tw-cdn.katalon.com/katalon-platform/configuration/bypass-limit.png" alt="How to set bypass limit" />


#### Result
After the set idle duration, users will see a prompt allowing them to extend their session — but only up to the specified number of times.

## Enable notification for Latest Katalon Studio Version

Stay updated with the latest releases of Katalon Studio.

1. Enable **Notify the latest Katalon Studio version**.
2. Click **Save Changes**.
<img src="https://tw-cdn.katalon.com/katalon-platform/configuration/notify-version.png" alt="Enable users to know the update" />

## Enable uploading Test Results to TestOps

Upload test results directly from Katalon Studio to Katalon TestOps for centralized tracking and analysis.

1. Enable **Upload test results to TestOps**.
2. Click **Save Changes**.
