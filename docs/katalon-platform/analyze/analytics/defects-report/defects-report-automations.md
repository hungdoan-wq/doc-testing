## What is a Defects Report automation?

<p xmlns="http://www.w3.org/1999/xhtml" className="p">Setting up a Defect report automation is the option to automate the linking of Jira defects to TestOps. It's useful for situations when there are multiple test results with the same issue that can be linked to a particular defect.</p> 
<p xmlns="http://www.w3.org/1999/xhtml" className="p">This makes the process of tracing defects easier. After executing a test run in TestOps, link your test results to an issue in Jira manually by using <span className="ph uicontrol">Report Defect</span>. See: <a className="xref" href="/katalon-platform/integrations/jira-integration/link-test-runs-to-jira-defects-in-katalon-testops">Link test runs  to Jira defects in Katalon TestOps</a>. </p> 
<p xmlns="http://www.w3.org/1999/xhtml" className="p">Then, create automation rules as shown below.</p> 
<p xmlns="http://www.w3.org/1999/xhtml" className="p"> </p>

## <a id="troubleshooting-9175" class="anchor_top_offset"/>Troubleshoot automation rules

<section xmlns="http://www.w3.org/1999/xhtml" className="section condition"><p className="p">The <span className="ph uicontrol">Save</span> button will remain greyed out  if there are unacceptable parameters within your automation rule. </p></section> 
<div xmlns="http://www.w3.org/1999/xhtml" className="bodydiv troubleSolution"><section className="section cause"><p className="p">There are unacceptable parameters within your automation rule. All form fields must pass validation before you can click <span className="ph uicontrol">Save</span>. Refer to the steps below for the following guidance in entering value for your automation rule.</p></section><section className="section remedy"><ol className="ol steps"><li className="li step"><span className="ph cmd">If your <span className="ph uicontrol">Rule Name</span> is blank or duplicated, enter a unique name with a maximum length of 255 characters. </span></li><li className="li step"><span className="ph cmd">If your <span className="ph uicontrol">Error Message</span> or <span className="ph uicontrol">Stack Trace</span> patterns are blank, enter a definition for your patterns using either the exact text or regular expression syntax. At least one of them must have a value with a minimum of 2 characters and a maximum of 2048.</span></li><li className="li step"><span className="ph cmd">If your Regex pattern for <span className="ph uicontrol">Error Message</span> or <span className="ph uicontrol">Stack Trace</span> is invalid, double-check your entry and refer to this document to learn more about it: <a className="xref j-external-link" href="https://www.postgresql.org/docs/current/functions-matching.html#FUNCTIONS-POSIX-REGEXP" target="_blank">Learn more about POSIX regular expressions here</a>. </span></li></ol></section></div>
