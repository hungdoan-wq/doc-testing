<p xmlns="http://www.w3.org/1999/xhtml" className="shortdesc"> This document shows you how to create a test suite with a test case. </p> 
<div xmlns="http://www.w3.org/1999/xhtml" className="section prereq p"><ul className="ul"><li className="li"><p className="p">You must have already created a test case. You cannot create a test suite without adding a test case to it. </p></li></ul></div>
<section xmlns="http://www.w3.org/1999/xhtml" className="section context">Create your test suite by clicking on <span className="ph uicontrol">+ Test Suite</span> within the Test Directory. </section> 
<ol xmlns="http://www.w3.org/1999/xhtml" className="ol steps"><li className="li step stepexpand"><span className="ph cmd">Go to <span className="ph menucascade"><span className="ph uicontrol">Tests</span><abbr title="and then"> &gt; </abbr><span className="ph uicontrol">Test Suites</span></span>  in the sidebar. The Test Directory  appears. </span></li><li className="li step stepexpand"><span className="ph cmd">Click on the <span className="ph uicontrol">+ Test Suite button</span>. The <span className="ph uicontrol">Create Test Suite dialog box</span> appears. </span><div className="itemgroup info"><img className="image" width={700} src={useBaseUrl("/a05811d7-4114-4311-8046-fa13ffac29f5/TO3B1_Create_Test_Suite_Dialog_Box.png")} alt="The create test suite dialog box in Katalon TestOps 3." /></div></li><li className="li step stepexpand"><span className="ph cmd">Enter the test suite name in the text box.</span></li><li className="li step stepexpand"><span className="ph cmd">Right below the name, click on the tag and select between Automated or Manual. </span></li><li className="li step stepexpand"><span className="ph cmd">Optionally, enter search filters to make it easier to find the test cases you want to add. </span><div className="itemgroup info"><ul className="ul"><li className="li"><strong className="ph b">Search</strong>: allows you to match text to the name of your test case.</li><li className="li"><strong className="ph b">Repository</strong>: allows you to specify from which repository you would like ot fetch your test cases from.</li><li className="li"><strong className="ph b">Type</strong>: allows you to filter test cases to only show the ones with your selected tag.</li><li className="li"><strong className="ph b">Status</strong>: allows you to filter test cases based on the working status you've ticked.</li><li className="li"><strong className="ph b">Author</strong>: Allows you to select test cases that have been authored by a specific individual.</li></ul><p className="p" /></div></li><li className="li step stepexpand"><span className="ph cmd">Upon identifying the test cases you'll add to the test suite, click on the <span className="ph uicontrol">Add (+) button</span> that's at the rightmost column of every test case row. </span><div className="itemgroup info"><div className="note note note_note"><span className="note__title">Note:</span> <ul className="ul"><li className="li"><p className="p">Currently, you cannot mix manual and automated tests as it could cause execution failures. </p></li></ul><p className="p" /></div></div></li><li className="li step stepexpand"><span className="ph cmd">Verify that your test cases have been added by going to <span className="ph uicontrol">Test Case List</span> and checking manually. </span></li><li className="li step stepexpand"><span className="ph cmd">Click <span className="ph uicontrol">Create</span>. </span></li></ol> 
<section xmlns="http://www.w3.org/1999/xhtml" className="section result">A toast notification confirms the successful creation of your test suite. Additionally, the newly created test suite appears in the list within the Test Directory page.</section>
