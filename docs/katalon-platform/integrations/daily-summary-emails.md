## What is the Daily Summary Email? 

<p xmlns="http://www.w3.org/1999/xhtml" className="p">This feature provides users with an overview of all test runs from user-specified projects through concise updates sent via email on a daily basis.    Users receive updates on the number of test cases, the outcomes of these tests, and general information on the most recent test runs.</p> 
<p xmlns="http://www.w3.org/1999/xhtml" className="p">These daily emails  ensure users can effortlessly keep track of their project's testing progress and performance with minimal manual intervention. </p>

## <a id="troubleshooting-4483" class="anchor_top_offset"/>Troubleshoot Daily Summary Emails

<p xmlns="http://www.w3.org/1999/xhtml" className="shortdesc"> </p> 
<section xmlns="http://www.w3.org/1999/xhtml" className="section condition"><p className="p">If you are not receiving any email, check that your settings or lack of activity aren't preventing it from going through. </p></section> 
<div xmlns="http://www.w3.org/1999/xhtml" className="bodydiv troubleSolution"><section className="section cause"><p className="p">You might have disabled this feature from your <span className="ph uicontrol">User Settings</span>. This blocks you from receiving any daily summary email even if you are   added to the user list in that project's settings. </p></section><section className="section remedy"><ol className="ol steps"><li className="li step stepexpand"><span className="ph cmd">Check your notifications' configurations and ensure that Daily Summary Report is toggled to the enabled setting. For a more detailed step-by-step on how to do so, see: <a className="xref" href="/katalon-platform/administer/settings/configure-email-notifications-in-testops">Configure email notifications in TestOps</a>. </span><div className="itemgroup info"><img className="image" src={useBaseUrl("/9eb9fb10-daad-11ee-9719-0242c7a41fd4/KP_Daily_Summary_Report_Troubleshooting.png")} alt="Toggle the daily summary report feature from user settings." /></div></li><li className="li step stepexpand"><span className="ph cmd">If the feature is confirmed as enabled but emails are still not received, it could be because the project hasn't had any testing activity for 28 straight days. An email like the one below will inform you that the daily reports have been paused. </span><div className="itemgroup info"><img className="image" src={useBaseUrl("/b43b6f00-df6c-11ee-b3a4-0242c7a41fd4/KP_Daily_Summary_Email_Inactive_28_Days.png")} alt="An email confirming that daily summary report has been disabled due to inactivity." /></div><div className="itemgroup info">Email notifications are automatically turned off for projects that are inactive for more than 28 days. However, once testing activity picks up again, email notifications will likewise recommence. </div></li></ol></section></div>
